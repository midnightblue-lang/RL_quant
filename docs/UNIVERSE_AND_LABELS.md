# 股票池与标签口径

本文件记录阶段 2 的实现口径。核心原则是：股票池只能使用调仓日当时可见的信息；未来收益只能作为标签，不能反向改变股票池。

## 股票池

每个月末交易日构建一次股票池：

1. 使用标准化日频行情面板。
2. 对每只股票统计截至当日的历史交易记录数。
3. 上市交易记录数少于 `min_listed_days` 的股票剔除。
4. 使用截至当日的过去 `liquidity_window` 个交易记录计算平均成交额。
5. 当日 OHLC 缺失、价格非正、成交量或成交额不可用的股票剔除。
6. 如果 `exclude_zero_volume=true`，当日零成交股票剔除。
7. 如果 `exclude_st=true`，名称包含 `ST` 或 `退` 的股票剔除。
8. 按过去窗口平均成交额降序排序，取 TopN。

注意：

- 当前实现的 `listed_days` 基于本地缓存中的历史交易记录数。如果缓存从 2015 年开始，老股票在样本初期会被保守地视为“记录不足”。后续扩全市场时，应优先接入上市日期和完整 A股交易日历来计算真实上市交易日数。
- 当前阶段的 ST 过滤依赖行情缓存中的名称字段。如果名称是当前名称而非历史名称，会有历史 ST 口径误差。
- 当前样本来自可下载股票数据，可能存在幸存者偏差。阶段 2 暂不重建完整历史上市/退市股票列表。
- 不使用下一交易日开盘、涨跌停或成交状态筛选股票池；这些属于后续回测执行约束。
- 当前行情配置使用 `qfq`。前复权历史价格可能随除权除息重新调整，后续进入因子研究时必须重新审查价格水平类因子的复权风险。

## 标签

未来 20 日收益标签使用市场交易日历：

```text
future_20d_return = close[t+20_market_trading_days] / close[t] - 1
```

如果目标交易日该股票没有收盘价，则不生成该样本标签。不要使用该股票自己的下一条可用记录替代目标日，否则停牌/缺失会导致标签向未来漂移。

`future_20d_rank` 在同一 `date` 的有效样本内做截面百分位排名。若传入股票池，则 rank 只在股票池样本中计算。

## 泄漏约束

- 修改调仓日之后的成交额，不应改变调仓日股票池。
- 修改标签窗口之后的价格，不应改变当前标签。
- 标签生成不得改变 `universe_monthly.parquet`。
- 股票池和标签分别缓存，便于独立审查。
